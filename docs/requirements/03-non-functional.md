# 03. 非機能要件 (Non-Functional Requirements)

機能ではなく品質特性に対する要件です。設計・実装上の判断はこれを根拠にしてください。

## ユーザビリティ

| ID | 要件 |
| --- | --- |
| NFR-00001 | 個人開発者が、初回セットアップから初回 Run 完了までを 5 分以内に到達できる |
| NFR-00002 | 主要操作 (登録・実行・結果確認) は単一の CLI から到達できる |
| NFR-00003 | エラー発生時、原因の切り分けに必要な情報 (どの Case が、どの理由で) が表示される |
| NFR-00004 | 結果出力は人間が読んだ瞬間にモデル順位が把握できる |

## 再現性

| ID | 要件 |
| --- | --- |
| NFR-00101 | 同一 Run 設定・同一環境で再実行した場合、集計値は決定的に再現する。LLM 応答の揺れは複数 Trial の統計集計で吸収する |
| NFR-00102 | Run の生成条件 (温度・seed 等) と provider 識別は結果に保存される |
| NFR-00103 | 生応答と正規化済み集計値の両方が保存される |

## 保守性・拡張性

| ID | 要件 |
| --- | --- |
| NFR-00201 | 新しい provider の追加は、評価ロジックや結果スキーマを変更せずに行える |
| NFR-00202 | 新しい scorer の追加は、既存 Task Profile に影響を与えずに行える |
| NFR-00203 | Task Profile および Case は、ツール本体とは独立して追加・更新できる |
| NFR-00204 | 結果スキーマには版を識別する情報が含まれ、後方互換性を判定できる |

## 移植性・依存

| ID | 要件 |
| --- | --- |
| NFR-00301 | macOS / Linux で動作する。Windows は v1 で必須としない |
| NFR-00302 | Python 標準ライブラリ中心の依存構成とし、外部依存は導入根拠を要件 ID と紐付けて記録する |
| NFR-00303 | provider のモデルダウンロード・常駐管理は本ツールの責務外とする |

## セキュリティ

| ID | 要件 |
| --- | --- |
| NFR-00401 | 認証情報は設定ファイルに平文で置かない。環境変数経由で読み込む |
| NFR-00402 | provider への通信は localhost を既定とし、外部送信を伴う動作は明示設定なしには行わない |

## 観測性

| ID | 要件 |
| --- | --- |
| NFR-00501 | Run の進行状況がユーザーに継続的に伝わる (進捗表示) |
| NFR-00502 | Run 失敗時、再試行や原因確認に必要な情報が結果ファイルとして残る |

## 性能

| ID | 要件 |
| --- | --- |
| NFR-00601 | 本ツール自体のオーバーヘッドは、各 Case の LLM 応答時間と比べて十分小さいこと (目安: 1 Trial あたり 100ms 未満) |
| NFR-00602 | 結果ファイル群は、典型的な個人利用 (10 モデル × 10 Task Profile × 5 Trial) で常識的な容量に収まること |

性能数値は v1 開発中に再評価し、必要に応じて改訂します。
