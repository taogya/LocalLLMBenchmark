# 01. 概要 (Overview)

要件定義の入口となる文書です。背景・目的・用語を共有し、以降の機能要件 (FUN-) と非機能要件 (NFR-) の前提を揃えます。

## 背景

ローカル LLM の選択肢は急速に増えており、同じ PC でも候補となるモデルが多数存在します。一方、Web 上で公開されているベンチマークは次のいずれかの理由で、個人ユーザーのモデル選定にそのまま使えません。

- 評価環境がユーザーの PC と異なる
- 評価タスクがユーザーの実用途と異なる
- 計測条件が公開されていない、または再現できない

結果として、個人開発者は自分の PC でモデルを総当たりするか、評判だけで選ぶことになります。

## 目的

| ID | 要求 |
| --- | --- |
| REQ-00001 | ユーザー自身の PC 上で、ユーザー自身が定義した用途に対するモデル比較を可能にする |
| REQ-00002 | 比較結果から「最適なモデル」をユーザーが選べる十分な情報を提供する |
| REQ-00003 | 同じ条件で何度実行しても同じ集計値が再現できる仕組みを提供する |
| REQ-00004 | ユーザーが用途を増やしたり候補モデルを増やすことを、低い学習コストで支援する |

## 想定ユーザー像

| ID | 説明 |
| --- | --- |
| REQ-00005 | ローカルで LLM を試している個人開発者。1 台の PC を前提とする |
| REQ-00006 | 自分の用途に近い prompt と期待出力を、自分で用意できる程度の技術力を持つ |

チーム共有・CI 統合・複数 PC 集約は v1 の対象外とします (詳細は [04-out-of-scope.md](04-out-of-scope.md))。

## 利用シーン

| ID | シーン |
| --- | --- |
| REQ-00007 | 新しいモデルを pull した直後に、既存候補と並べて比較したい |
| REQ-00008 | PC のスペックや構成を変更した後に、最適モデルが変わったかを再確認したい |
| REQ-00009 | 自分の用途を新しく追加し、それに対する各モデルの適性を知りたい |

## 用語

| 用語 | 定義 |
| --- | --- |
| Task Profile | ユーザーの用途を表す単位。1 つ以上の Case と評価契約を含む |
| Case | Task Profile に属する 1 件の入力と期待出力 |
| Model Candidate | 比較対象のモデル。provider と provider 上の識別子を持つ |
| Run | 候補モデル群を Task Profile 群に対して実行した一回分の計測単位 |
| Trial | 同一 Case の繰り返し実行 1 回分。Run は複数 Trial を含む |
| Quality Score | 決定的な scorer による評価値 (0..1 または bool) |
| Performance Metric | provider が返す応答時間とトークン消費 |
| Ranking | 品質と性能を統合してモデルを順位付けした表 |

## 想定外の利用

本ツールは次の用途を想定しません。利用しても結果は誤解を招く可能性があります。

- 公開向けのモデル評価レポート作成
- モデル提供者間の優劣を主張する根拠
- ベンチマーク値そのものを SLO に組み込むこと

これらが必要な場合は、別途設計された公開向けベンチマークの利用を推奨します。
