# 開発・実行環境

本書は v1 で実装着手するための環境前提と、外部依存導入の運用ルールを定めます。具体的なパッケージ一覧は網羅しません。導入是非は実装 task 単位で個別判断します。

関連既存 ID: NFR-00301, NFR-00302, NFR-00402, ARCH-00006, OOS-00005, OOS-00006

## 対応 OS

| 区分 | 対応 |
| --- | --- |
| 必須 | macOS / Linux (NFR-00301) |
| 任意 | Windows (v1 で必須としない。動作報告は歓迎) |

## Python バージョン

- 既定: Python 3.13
- 採用根拠:
  - 本プロジェクトは外部依存を最小限にし、Python 標準ライブラリを中心に実装する方針 (NFR-00302, ARCH-00006) を採る。Python 3.13 は Configuration & Storage 層で想定される TOML 読込 (`tomllib`) ・決定的な統計処理 (`statistics`) ・サブプロセス制御 ・型ヒント拡張が標準ライブラリで提供されるため、依存追加なしで v1 の要件を成立させやすい。
  - macOS / Linux (NFR-00301) で長期サポート上の主流バージョンとして入手可能で、個人ユーザーが pyenv 等で導入しやすい。

新版 Python への追従は v1.1 以降で再評価します。

## 仮想環境

- 推奨: pyenv で Python 3.13 を導入し、プロジェクトルート直下の `.venv` を venv で作成
- 強制ではない。Conda / uv / Rye 等の利用は妨げない。再現性は依存方針 (下記) で担保する

## 依存方針

| 区分 | 方針 |
| --- | --- |
| 既定 | Python 標準ライブラリで実装する (ARCH-00006, NFR-00302) |
| 例外 | 標準ライブラリで合理的に実現できない場合に外部依存を検討する |
| 記録 | 外部依存を導入する task は「導入根拠となる要件 ID (REQ- / FUN- / NFR-)」を本文に明記する |
| 撤去容易性 | 外部依存は使用箇所をコンポーネント境界で限定し、差し替え/撤去が単一箇所で済むよう保つ |
| 版固定 | 導入時に上限/下限の方針を実装 task で明記する |

具体的な依存パッケージ (HTTP クライアント、TOML ローダ等) は実装 task で個別に判断します。本書では候補名を列挙しません。

## Provider 前提

| ID 参照 | 前提 |
| --- | --- |
| NFR-00402 | provider への通信は localhost を既定。外部送信は明示設定なしに行わない |
| OOS-00005 | モデルの自動 download / pull は本ツールの責務外 |
| OOS-00006 | provider プロセスの起動・停止は本ツールの責務外 |

v1 のサポート provider は Ollama のみ (PVD-00401)。Ollama が localhost で起動済みかつ対象モデルが pull 済みであることはユーザー責務とします。

## 開発時に推奨される確認

- 設計確定済み task の Status (PROG-00103) を実装着手前に確認する
- 変更前に対応 ID の参照範囲を `docs/` `progress/` で洗い出す (TRC-00102, TRC-00103)
- ローカルで `check` (CLI-00105) 相当の整合性検証が成功するまで Run 着手しない
